hexo是什么

hexo是基于node.js的静态站点生成器

能够将含有结构化元数据的文本（通常使用markdown）在构建期转换为纯静态的HTML/CSS/JS文件，并部署到静态托管环境

（通俗来讲就是：把一堆markdown当成源码编译成网站）

hexo命令

1.hexo init

在某个目录下初始化，生成一个新的 Hexo 站点。

生成标准目录结构（_config.yml、package.json、scaffolds、source、themes 等），并自动安装依赖。

等价于“创建项目骨架”，只需执行一次。

2.hexo new [layout] </p> <p>layout是post时，titile应为文章名，作用是在soutce/_post下创建新文件title</p> <p>layout是page时，title应为索引头类型，比如about，categories，tag，对应网站上home同级别的索引头</p> <p>对应source下的文件夹about，categories等。</p> <p>3.<strong>hexo clean</strong></p> <p>清理生成缓存和静态文件（删除 public/ 和缓存目录）。</p> <p>4.<strong>hexo generate (hexo g)</strong></p> <p>将 Markdown 源文件编译为静态 HTML 文件,输出到public/ ，构建产物生成</p> <p>5.<strong>hexo server (hexo s)</strong></p> <p>启动本地 HTTP 服务预览站点。</p> <p>在本地用网站<a href="http://localhost:4000浏览">http://localhost:4000浏览</a></p> <p>用<strong>hexo clean&&hexo g&&hexo s</strong>快速启动</p> <p>hexo s相当于本地调试服务器，不涉及真实部署。</p> <p>6.<strong>hexo deploy (hexo d)</strong></p> <p>事先安装好hexo-deployer-git后，将public/中的静态文件部署到远端（github page)</p> <p>可以用<strong>hexo clean&&hexo g&&hexo d</strong>快速部署</p>
